# Contributing to ShopSphere-DevOps

Thank you for your interest in contributing to ShopSphere!

This project is built in public to demonstrate modern software engineering and DevOps practices. Contributions of all sizes are welcome.

## How to Contribute

1. Fork the repository.
2. Create a new branch from `main`.
3. Make your changes.
4. Commit your changes using the Conventional Commits specification.
5. Push your branch to your fork.
6. Open a Pull Request.

## Branch Naming

Use descriptive branch names such as:

* `feature/add-user-profile`
* `fix/login-validation`
* `docs/update-readme`
* `chore/update-dependencies`

## Commit Messages

Examples:

```text
feat(auth): add JWT authentication

fix(product): resolve pagination issue

docs(readme): update project roadmap

chore(repo): clean project structure
```

## Pull Requests

Please ensure that:

* Your code builds successfully.
* Documentation is updated when necessary.
* Pull requests are focused on a single change.
* Commit history is clean.

## Reporting Issues

If you've found a bug or have a feature request, please open an issue using the provided GitHub issue templates.

## Code of Conduct

By participating in this project, you agree to abide by the project's Code of Conduct.
